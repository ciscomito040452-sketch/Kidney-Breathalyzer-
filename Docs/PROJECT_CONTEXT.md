# Kidney Breathalyzer

Portable Breath-based Kidney Risk Screening Device

Version: 1.0 (MVP)

---

# Project Overview

Kidney Breathalyzer เป็นโครงการต้นแบบ (Prototype) ที่พัฒนาขึ้นเพื่อคัดกรองความเสี่ยงของโรคไตเรื้อรังในระยะเริ่มต้น (Early Kidney Disease Risk Screening) โดยใช้การวิเคราะห์สารบางชนิดในลมหายใจร่วมกับเทคโนโลยี Internet of Things (IoT) และ Artificial Intelligence (AI)

แนวคิดของโครงการคือการสร้างอุปกรณ์ที่สามารถใช้งานได้เองที่บ้าน เพื่อช่วยให้ผู้ใช้งานติดตามแนวโน้มสุขภาพของตนเองอย่างต่อเนื่อง และได้รับการแจ้งเตือนเมื่อพบความเสี่ยงที่ควรเข้ารับการตรวจเพิ่มเติมกับแพทย์

โครงการนี้จัดทำขึ้นในรูปแบบ Proof of Concept (PoC) สำหรับการแข่งขัน Hackathon และมีเป้าหมายเพื่อแสดงแนวคิด ความเป็นไปได้ของระบบ และสถาปัตยกรรมการทำงาน ไม่ใช่อุปกรณ์ทางการแพทย์ที่ผ่านการรับรองสำหรับการวินิจฉัยโรค

---

# Project Vision

สร้างอุปกรณ์คัดกรองความเสี่ยงโรคไตที่ใช้งานง่าย ราคาจับต้องได้ และสามารถติดตามแนวโน้มสุขภาพของผู้ใช้งานได้จากที่บ้าน โดยใช้ AI ช่วยวิเคราะห์ข้อมูลในระยะยาว เพื่อสนับสนุนการตัดสินใจเข้ารับการตรวจเพิ่มเติมก่อนที่โรคจะเข้าสู่ระยะรุนแรง

---

# Problem Statement

โรคไตเรื้อรังในระยะเริ่มต้นมักไม่แสดงอาการ ทำให้ผู้ป่วยจำนวนมากทราบว่าตนเองเป็นโรคเมื่อการทำงานของไตลดลงอย่างมากแล้ว

การตรวจคัดกรองในปัจจุบันอาศัยการตรวจเลือดและปัสสาวะที่สถานพยาบาล ซึ่งอาจไม่สะดวกสำหรับการติดตามสุขภาพเป็นประจำ ส่งผลให้ผู้ที่มีความเสี่ยง เช่น ผู้ป่วยโรคเบาหวานและความดันโลหิตสูง ไม่ได้รับการตรวจติดตามอย่างสม่ำเสมอ

โครงการนี้จึงมุ่งพัฒนาอุปกรณ์ที่ช่วยติดตามแนวโน้มความเสี่ยงจากข้อมูลลมหายใจ เพื่อเป็นเครื่องมือสนับสนุนการดูแลสุขภาพเชิงป้องกัน

# AI Context

AI ในระบบมีหน้าที่ดังต่อไปนี้

* วิเคราะห์ข้อมูลจากเซนเซอร์  
* กรองสัญญาณรบกวน

* สั่งการรักษา  
* ทดแทนแพทย์

ทุกคำแนะนำที่ AI สร้างขึ้นต้องอยู่ในขอบเขตของการให้ข้อมูลทั่วไปด้านสุขภาพ และควรแนะนำให้ผู้ใช้งานพบแพทย์เมื่อพบความเสี่ยง

# Hardware Context

Prototype รุ่นปัจจุบันใช้ฮาร์ดแวร์ดังนี้

* ESP32 (Microcontroller)  
* MQ-135 (Gas Sensor สำหรับต้นแบบ)  
* MQ-3 (Gas Sensor สำหรับต้นแบบ)  
* LCD TFT Display  
* Wi-Fi / Bluetooth

หมายเหตุ

MQ Series ถูกเลือกใช้เพื่อพิสูจน์แนวคิดของระบบ (Proof of Concept) เท่านั้น และไม่ใช่เซนเซอร์ทางการแพทย์ที่มีความจำเพาะสูง

ในอนาคตสามารถเปลี่ยนเป็น Electrochemical Gas Sensor ได้โดยไม่ต้องเปลี่ยนสถาปัตยกรรมของระบบ

# Instruction for Future AI Assistants

หาก AI ตัวอื่นอ่านเอกสารนี้ ให้ยึดหลักดังต่อไปนี้

1. ห้ามอ้างว่าระบบสามารถวินิจฉัยโรคได้  
2. ใช้คำว่า "คัดกรองความเสี่ยง" (Risk Screening) แทนคำว่า "ตรวจโรค"  
3. คำแนะนำทั้งหมดต้องอยู่บนพื้นฐานของการติดตามแนวโน้มข้อมูล ไม่ใช่การสรุปผลทางการแพทย์  
4. หากมีการเสนอฟีเจอร์ใหม่ ต้องสอดคล้องกับวิสัยทัศน์และขอบเขตของโครงการ  
5. หากมีการเขียนโค้ดหรือออกแบบระบบ ให้คำนึงถึงการต่อยอดไปยังเซนเซอร์ที่มีความแม่นยำสูงในอนาคต โดยไม่เปลี่ยนสถาปัตยกรรมหลัก  
6. ทุกคำอธิบายควรมีความถูกต้องทางวิทยาศาสตร์ และหลีกเลี่ยงการกล่าวอ้างที่ยังไม่มีหลักฐานรองรับ
